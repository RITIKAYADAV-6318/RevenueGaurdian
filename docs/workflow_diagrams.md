# Revenue Guardian: Workflow Diagrams

This document contains a set of Mermaid workflow diagrams visualizing the operational processes, approval lifecycles, and automated playbooks of the **Revenue Guardian** platform.

---

## 1. Autonomous Daily Auditing Workflow
Visualizes the daily schedule loop managed by the background scheduler.

```mermaid
flowchart TD
    %% Scheduler Start
    Start([08:00 AM Trigger]) --> Wake[Scheduler Wakes Up]
    
    %% Phase 1: Data Gathering
    Wake --> Fetch_CRM[MCP: Fetch CRM Leads & Opportunities]
    Wake --> Fetch_Email[MCP: Fetch Gmail Threads]
    Wake --> Fetch_Cal[MCP: Fetch Calendar Events & Availability]
    
    %% Phase 2: Agent Core
    Fetch_CRM --> Run_Orch[Execute RevOpsOrchestrator]
    Fetch_Email --> Run_Orch
    Fetch_Cal --> Run_Orch
    
    Run_Orch --> Run_Predict[Run Revenue Prediction Agent]
    Run_Predict --> Run_Recovery[Run Recovery Strategy Agent]
    Run_Recovery --> Run_Summary[Run Executive Summary Agent]
    
    %% Phase 3: Action & Storage
    Run_Summary --> Save_DB[(Save Run Output to SQLite)]
    Run_Summary --> Save_Doc[Save Markdown Report in docs/reports/]
    Run_Summary --> Update_Dash[MCP: Update Dashboard Widgets]
    Run_Summary --> Send_Slack[MCP: Send Morning Briefing to Slack #revops-alerts]
    
    %% Sleep
    Save_DB --> Sleep[Calculate Seconds until 08:00 AM Tomorrow]
    Save_Doc --> Sleep
    Update_Dash --> Sleep
    Send_Slack --> Sleep
    
    Sleep --> Wait[Asynchronous Sleep]
    Wait -->|Timer Expires| Start
```

## 2. Human-in-the-Loop (HITL) Approval Lifecycle
Illustrates the governance process protecting financial and client-facing operations.

```mermaid
flowchart TD
    %% Alert Generated
    Start([Alert Generated by Orchestrator]) --> Queue[Action Added to Pending Approvals Queue]
    Queue --> Dash[Displayed on Streamlit Dashboard]
    
    %% User Review
    Dash --> User_Review{Manager Reviews Action}
    
    %% Decision 1: Reject
    User_Review -->|Reject| Delete[Mark Action as 'Rejected']
    Delete --> Log_Reject[Log Rejection in Audit DB]
    Log_Reject --> End_Reject([Workflow Terminated])
    
    %% Decision 2: Modify
    User_Review -->|Edit / Modify| Modify[Modify Email Body or Invoice Amount]
    Modify --> Save_Draft[Save Modified Draft in DB]
    Save_Draft --> User_Review
    
    %% Decision 3: Approve
    User_Review -->|Approve| Verify_Role{User Role == 'CFO' or 'RevOps_Manager'?}
    
    Verify_Role -->|No| Block[Block Action & Show 403 Error]
    Block --> User_Review
    
    Verify_Role -->|Yes| Execute[Execute Action via MCP Server]
    
    %% Execution
    subgraph Execution_Layer [MCP Execution]
        Execute -->|Email Action| Send_Gmail[MCP: Send Gmail Message / Create Draft]
        Execute -->|Billing Action| Stripe_Adj[MCP: Create Stripe Invoice Overage]
        Execute -->|CRM Action| CRM_Up[MCP: Update Opportunity Stage]
    end
    
    Send_Gmail --> Log_Success[Log Success in Audit DB]
    Stripe_Adj --> Log_Success
    CRM_Up --> Log_Success
    
    Log_Success --> End_Approve([Action Successfully Executed])
```

## 3. Telemetry-Based Churn Recovery Playbook
Visualizes the automated playbook triggered by a sudden drop in product usage.

```mermaid
flowchart TD
    %% Telemetry Spike
    Start([Usage Sentinel Detects >30% Drop in Active Usage]) --> Alert_Orch[Alert Orchestrator]
    
    %% Context Gathering
    Alert_Orch --> Get_CRM[Query CRM: Retrieve Account Owner & Contract Value]
    Alert_Orch --> Get_Emails[Query Gmail: Check Recent Customer Sentiment]
    
    %% Analysis
    Get_CRM --> Run_Predict[Calculate Churn Probability & Revenue at Risk]
    Get_Emails --> Run_Predict
    
    Run_Predict --> Decision{Is Churn Risk > 50%?}
    
    %% High Churn Risk Path
    Decision -->|Yes| High_Risk[High Churn Risk Playbook]
    High_Risk --> Draft_Email[Sales Assistant Drafts Urgent CS Outreach Email]
    High_Risk --> Slack_Alert[MCP: Dispatch Urgent Alert to Slack #revops-alerts]
    High_Risk --> Escalate[Recovery Agent Drafts Account Manager Escalation Node]
    
    %% Low Churn Risk Path
    Decision -->|No| Low_Risk[Low Churn Risk Playbook]
    Low_Risk --> Add_Task[CRM Agent: Add 'Check-in' Task to CRM Queue]
    
    %% Merging to HITL
    Draft_Email --> HITL[Queue for Manager Approval]
    Slack_Alert --> HITL
    Escalate --> HITL
    Add_Task --> HITL
    
    HITL --> End([Playbook Awaiting Execution])
```

## 4. Contract Ingestion & Billing Reconciliation
Illustrates the workflow for onboarding a new contract and verifying billing setup.

```mermaid
flowchart TD
    %% Ingestion
    Start([User Uploads Contract PDF]) --> OCR[Extract Text via Gemini Multimodal]
    OCR --> Parser[Auditor Agent: Parse Key Terms]
    
    %% Extraction
    subgraph Extraction [Extracted Metadata]
        Parser --> Ext_Client[Customer Name]
        Parser --> Ext_Tier[Pricing Tier / Flat Fee]
        Parser --> Ext_Limit[Usage Limits]
        Parser --> Ext_Date[Effective & Renewal Dates]
    end
    
    %% Save to DB
    Ext_Client --> Save_Contract[(Save Contract Terms to SQLite)]
    Ext_Tier --> Save_Contract
    Ext_Limit --> Save_Contract
    Ext_Date --> Save_Contract
    
    %% Reconcile
    Save_Contract --> Query_Stripe[MCP: Fetch Active Stripe Subscription]
    Query_Stripe --> Reconcile{Match: Contract Tier == Stripe Subscription?}
    
    %% Match Decisions
    Reconcile -->|Yes| Match_OK[Log Match Success in Audit DB]
    Match_OK --> End_Success([Account Verified & Active])
    
    Reconcile -->|No| Discrepancy[Generate Billing Discrepancy Alert]
    Discrepancy --> Draft_Billing[Sales Assistant Drafts Stripe Subscription Update]
    Draft_Billing --> Notify[Notify Slack #revops-alerts]
    Notify --> HITL[Queue Billing Update for CFO Approval]
    
    HITL --> End_HITL([Reconciliation Pending Approval])
```
